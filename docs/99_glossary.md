# 用語集（Glossary）

- Issue: 課題チケット。やることの定義。
- Pull Request（PR）: 変更提案。差分を見てレビューし、OKならマージする。
- ブランチ: 変更作業の「枝」。mainを直接いじらないために使う。
- マージ: 変更をmainに取り込むこと。
- コミット: 変更履歴の1単位。巻き戻し可能にするための記録。
- Runbook: セットアップや起動、確認手順をまとめた実行手順書。
- Status: 現状と次の一手をまとめた要約。中断からの復帰点。
- 受け入れ条件（Acceptance Criteria）: これが満たされたらOK、という合格条件。
- .env: 環境変数を入れるファイル。APIキーなどを置くが、Gitに入れない。
- .gitignore: Gitの管理対象から除外する指定ファイル。
