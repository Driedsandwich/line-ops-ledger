# セキュリティ（Security）

## 絶対に守ること
- APIキーやトークンは「パスワード以上に重要」な情報として扱う
- コードに直接書かない
- `.env` に置き、Gitにコミットしない
- Issue/PR本文やスクリーンショット、ログ貼り付けにも書かない
- 漏えいが疑われたら、キーを無効化して再発行する（ローテーション）

## リポジトリ運用での最低ライン
- `.gitignore` に `.env` と依存フォルダ（例: node_modules/）等を含める
- `.env.example` は変数名だけを置き、値は入れない
- PRテンプレのチェックリストで「機密情報なし」を毎回確認する
- 可能ならGitHub側の機密情報保護機能を有効にする

## 「外部に送ってよい情報」の境界
- docs/01_requirements.md の「データの機密度」に書いた範囲を超えて、外部AI/APIに送らない
- 迷う場合は送らない。代替案としてダミーデータを作る

## 漏えい時の行動（最小手順）
1. 影響範囲を仮決めする（どのキーか、どこに出たか）
2. 該当キーを無効化し、新しいキーを発行する
3. `.env` やコード、Issue/PRの履歴に残っていないか確認し、必要なら取り除く
4. 再発防止のために判断ログ（docs/04_decisions.md）に記録する
