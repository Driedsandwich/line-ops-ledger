# 00_START_HERE（AI用：作業開始ガイド）

このリポジトリは **AIだけが中身（ファイル）を読み書き**します。
人間（プロダクトオーナー）は、GitHubのIssue/PRで自然言語の指示と承認を行うだけです。

## AIの最初の3ステップ（必須）
1. **ルールを読む**
   - `AGENTS.md`（または同内容の `CLAUDE.md`）を最初に読む
2. **現状を読む**
   - `docs/03_status.md` を読む（今動くもの / 次の3つ）
   - 初回なら `docs/00_goal.md` と `docs/01_requirements.md` を読む
3. **不足情報を質問し、作業計画を出す**
   - 不明点がある場合は **仮定せず** Issueコメントで質問する
   - その後、**作業計画（箇条書き）** を提示し、承認を得てから実装する

## Bootstrap Issue の場合（必須）
Bootstrap Issue（`00: プロジェクト起動`）では、**最初のPRは台帳（docs）を確定させるだけ**にします（コード実装はしない）。

このPRでやること:
- Issue本文とコメントのQ&Aを根拠に、次を埋める/初期化する
  - `docs/00_goal.md`
  - `docs/01_requirements.md`
  - `docs/02_runbook.md`（前提だけでも埋める）
  - `docs/03_status.md`（「いま動くもの」は未実装と明記し、次の3つを決める）
- 不足があれば **仮定せず** Issueコメントで質問する

このPRがマージされてから、機能実装のIssue/PRに進む。

## 実装の進め方（標準）
- 作業はブランチで行い、Pull Requestを作成する（`main` へ直pushしない）
- 1PRは小さくする（無関係な変更を混ぜない）
- PRには以下を必ず含める
  - 変更概要
  - 確認手順（誰でも再現できる）
  - docsの更新（最低限 `docs/02_runbook.md` と `docs/03_status.md`）

## ドキュメント更新（最低ライン）
- `docs/02_runbook.md`: **動かし方**（セットアップ/起動/最短確認）を更新
- `docs/03_status.md`: **いま動くもの** と **次の3つ** を更新（PRマージごと）
- `docs/04_decisions.md`: 重要な判断があった場合だけ更新
- `docs/05_ai_log.md`: 1PRにつき1行の要約を追記（機密を貼らない）

## 最初に切るIssue（推奨）
- Issueテンプレ `00: プロジェクト起動（Bootstrap）` を使い、目的と受け入れ条件を固める
