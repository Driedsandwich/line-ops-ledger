# 回線運用台帳

ローカル優先・暗号化前提の PWA として、回線・期限・証跡・特典を安全に管理するためのリポジトリです。

## 現在できること

### ダッシュボード (`/`)
- データ 0 件時の初回ガイド（回線追加 / 確認用サンプルデータ投入 / 履歴確認 / バックアップ復元）
- 危険案件サマリー（期限超過件数）
- 契約終了が近い回線アラート（30日以内、最大5件）
- 長期未活動の回線カード（90日基準、最大5件）＋「活動を記録」クイックリンク
- 近日期限集計（1〜3日 / 4〜7日 排他表示）
- 状態別件数（利用中 / 解約予定 / 登録総数）
- 月額費用サマリー（設定済み回線の合計）
- 通知方針サマリー・通知対象件数・通知理由別件数
- 通知対象回線一覧 → `/lines` 理由付き遷移

### 主台帳 (`/lines`)
- 回線の追加・編集・削除・Undo（Ctrl+Z / ⌘Z）
- データ 0 件時の空状態ガイド（回線フォーム / 確認用サンプルデータ投入 / バックアップ復元 / 履歴ページ）
- 電話番号全文保存・原則マスク表示
- 回線種別 / 月額費用 / 次回確認日 / 契約開始日 / 契約終了日
- 契約者 / 使用者 / 支払方法 / プラン / 端末 / 契約名義メモ
- キーワード・契約状態・回線種別による絞り込み
- 並び替え（次回確認日 / 月額費用 / 作成日時 / 最終活動日）
- URLパラメータで並び順を指定可（`?sort=latestActivityAsc` など）
- 複数選択・一括ステータス変更・一括削除
- 行ごとの詳細表示の開閉
- 行ボタンから「活動を記録」クイックアクション

### 契約履歴・活動ログ (`/lines/history`)
- 履歴エントリの登録・編集・削除（電話番号単位）
- データ 0 件時の空状態ガイド（履歴フォーム / 確認用サンプルデータ投入 / 回線一覧 / バックアップ復元）
- 1件の履歴に複数の活動ログ
- 活動種別のクイック選択ボタン（よく使う種別 / 定義済み種別）
- 活動種別に応じた活動メモ候補（種別別の頻出文言 + fallback 候補）
- 活動メモのクイック候補（固定候補 / 定型候補 / 最近使った文言 / 非表示候補）
- 現在の活動メモ文言をその場で custom 候補として追加・削除
- 候補は重複表示を避けて整理され、現在値と完全一致する候補は選択中表示
- `固定候補` / `非表示候補` / 候補管理全体を 1 操作で初期化可能
- 電話番号に一致する主台帳候補 / 直近履歴候補のワンタップ反映
- 活動日のクイック入力（今日 / 契約開始日 / 前回活動日）
- 活動記録保存後に「次回確認日を更新しますか？」サジェスト（活動日+設定日数）
- 電話番号単位のタイムライン（期間切替・表示対象切替）
- 履歴 JSON エクスポート / インポート
- 主台帳＋契約履歴の統合バックアップエクスポート / インポート

### 設定 (`/settings`)
- `/settings/storage` / `/settings/backup` / `/settings/notifications` / `/settings/activity-types` のサブルート導線
- 通知方針設定（対象期限・再通知ポリシー）
- 永続ストレージ状態・schemaVersion・保存件数・最終更新時刻
- 統合バックアップのエクスポート / インポート

## セットアップ

```bash
npm install
npm run dev
```

## 開発用バッジ

作業ブランチを視認するため `.env.local`（gitignore済み）に設定する：

```
VITE_DEV_LABEL="DEV / feat/xxx / PR #NNN"
```

未設定時はバッジ非表示。ファイルはコミット不要・ブランチ変更のたびに手動更新。

## GitHub Actions

- `Repo sanity`: 最低限のファイル存在確認
- `CI`: `npm ci` → `npm run check` → `npm run build`
- `main` 保護: PR 必須 / approval 任意 / required check `check-and-build`

## 確認手順

1. `npm run dev` でサーバー起動
2. `/` でダッシュボードの各カードが表示されることを確認
3. localStorage 空の状態で、`/` に初回ガイド、`/lines` と `/lines/history` に空状態導線が出ることを確認
4. 初回ガイドの `確認用サンプルデータを読み込む` で、主台帳と履歴が即時反映されることを確認
5. `/lines` で回線の追加・編集・検索が動作することを確認
6. `/lines/history` で履歴記録・タイムライン・契約候補反映・活動種別 / 活動メモ / 活動日のクイック入力が動作することを確認
7. `活動メモ` 候補の重複が整理され、現在値と完全一致する候補が選択中表示になることを確認
8. `活動メモ` 候補の `固定` / `固定解除` / `非表示` / `戻す` が再読み込み後も保持されることを確認
9. `この文言を候補に追加` で custom 候補が増え、`削除` で消せることを確認
10. `固定候補をクリア` / `非表示候補をクリア` / `候補管理を初期化` が動き、再読み込み後も維持されることを確認
11. `/settings/storage`、`/settings/backup`、`/settings/notifications`、`/settings/activity-types` が直接開けることを確認
12. `npx tsc --noEmit` でビルドエラーがないことを確認
13. PR では GitHub Actions の `CI` workflow が成功していることを確認

## 重要方針

- 秘密情報は repo に保存しない
- 暗号化本体・通知本実装・IndexedDB 本体は段階的に進める
- AI が実装ドライバー、人間がプロダクトオーナー（Issue/PR で自然言語指示）
