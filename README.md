# AI主導開発テンプレートリポジトリ

このリポジトリは、**人間（非エンジニア）が自然言語で指示**し、**AIが実装を先導**する開発を前提にした「作業台帳＋ガードレール」テンプレートです。

重要: **リポジトリ内のファイルを読む/編集するのはAIのみ**です。  
人間は GitHub の **Issue/PRで指示と承認**を行います（ファイル編集はしない）。

最優先は次の3つです。
- 事故を避ける（特に機密情報の流出）
- 途中で止まっても再開できる（10分で状況を復元できる）
- 後から保守できる（判断の理由と手順が残る）

## 使い方（1プロダクト = 1リポジトリ）
1. GitHubでこのテンプレートから **新しいリポジトリを作成**する（プロダクトごとに複製）
2. 人間が Issue「00: プロジェクト起動（Bootstrap）」を作成し、目的と受け入れ条件を自然言語で書く
3. AIが `00_START_HERE.md` と `AGENTS.md` を読み、不足情報を質問 → 作業計画提示 → **最初のPRで docs を埋めて台帳を確定**する（Bootstrap PR。コード実装はしない）
4. 人間がPRをレビューしてマージする（目的/確認手順/機密情報の混入がないか）
5. Bootstrap PRのマージ後、AIが機能実装のIssue/PRを進める（PRごとに `docs/02_runbook.md` と `docs/03_status.md` を更新する）

## 初回だけやること（5分）
- GitHub側の設定で `main` を保護する（推奨）
  - PR経由、`Repo sanity` を必須チェックにする
- `.github/CODEOWNERS` を使う場合は、コメントを外して自分のGitHubユーザー名に置き換える
  - 使わない場合は、そのままでよい（何も強制しない）

## AIの開始地点
- `00_START_HERE.md`: 作業開始ガイド（必読）
- `AGENTS.md` / `CLAUDE.md`: AIの運用ルール（同内容）

## 重要ファイル（AIが更新）
- docs/00_goal.md: 目的と成功条件
- docs/01_requirements.md: 入出力、制約、受け入れ条件、確認手順
- docs/02_runbook.md: セットアップ、起動、最短確認手順（未来の自分のため）
- docs/03_status.md: いま動くもの、未実装、次の3つ（最優先）
- docs/04_decisions.md: 重要な判断と理由
- docs/05_ai_log.md: 1PR=1行の要約ログ（機密を貼らない）
- docs/06_security.md: セキュリティ運用の最低ライン

## prompts/ について
- `prompts/` は **AIが自己参照する作業テンプレ**です（人間がコピペして使う前提ではありません）。
- AIは必要に応じて `prompts/00_context-pack.md` 等を参照し、質問や計画の抜けを防ぎます。

## テンプレートのバージョン
- このテンプレートの版は `TEMPLATE_VERSION` に記録しています。
- 各プロジェクト側も `TEMPLATE_VERSION` を残し、いつの型で始めたかを追跡します。

## セキュリティ最重要ルール
- `.env` やAPIキーなどの機密情報は絶対にコミットしない
- `.env.example` は変数名だけを置き、値は入れない
- 漏えいが疑われたら、キーを無効化して再発行する（ローテーション）

詳細は `docs/06_security.md` を参照してください。
