# 回線運用台帳

ローカル優先・暗号化前提の PWA として、回線・期限・証跡・特典を安全に管理するためのリポジトリです。

## 現在できること
- `/` で保存済み回線の実データから危険案件と近日期限を集計表示できる
- 危険案件が 0 件でも `問題なし` を表示できる
- `/lines` で回線ドラフトを追加・編集・削除できる
- `/lines` で次回確認日を保持できる
- 直前の追加・更新・削除を `操作を戻す` または `Ctrl+Z` / `⌘Z` で戻せる
- `/settings` で永続ストレージ状態に加えて schema version / 保存件数 / 最終更新時刻を確認できる
- manifest / service worker / offline shell を確認できる

## セットアップ
```bash
npm install
npm run dev
```

## 確認手順
1. ブラウザで `/lines` を開く
2. 回線名、キャリア、契約状態、次回確認日を入力して保存する
3. `/` を開いて危険案件サマリーと近日期限が更新されることを確認する
4. `/settings` を開いて schema version、保存件数、最終更新時刻が表示されることを確認する
5. `/lines` の追加・編集・削除・Undo が引き続き動くことを確認する

## 重要方針
- 秘密情報は repo に保存しない
- docs-only では終えず、毎回確認可能な挙動差分を出す
- 暗号化本体・通知本実装・IndexedDB 本体は段階的に進める
