# 回線運用台帳

ローカル優先・暗号化前提の PWA として、回線・期限・証跡・特典を安全に管理するためのリポジトリです。

## 現在できること
- `/` でダッシュボード骨組みを表示
- 危険案件が 0 件でも `問題なし` を表示
- `/lines` で回線ドラフトを追加・編集・削除できる
- ページ再読み込み後も回線ドラフトが残る
- `/settings` で永続ストレージ状態を確認
- `persist を要求する` ボタンで永続化要求の結果を確認
- manifest / service worker / offline shell を確認

## セットアップ
```bash
npm install
npm run dev
```

## 確認手順
1. ブラウザで `/` を開く
2. 危険案件サマリーが `0件 / 問題なし` で表示されることを確認
3. `/lines` を開く
4. 回線名、キャリア、契約状態を入力して保存する
5. 一覧に新しい回線が表示されることを確認する
6. 保存済み回線を編集し、一覧へ反映されることを確認する
7. 保存済み回線を削除し、一覧から消えることを確認する
8. ページを再読み込みし、保存内容が残ることを確認する
9. `/settings` を開く
10. 永続ストレージ状態と容量目安が表示されることを確認
11. `persist を要求する` を押して結果メッセージが表示されることを確認
12. ブラウザの Application タブで manifest / service worker を確認

## 現在の優先事項

## 重要方針
- 秘密情報は repo に保存しない
- docs-only では終えず、毎回確認可能な挙動差分を出す
- 暗号化本体・通知本実装・IndexedDB 本体は段階的に進める
