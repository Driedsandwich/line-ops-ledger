# 回線運用台帳

ローカル優先・暗号化前提の PWA として、回線・期限・証跡・特典を安全に管理するためのリポジトリです。

## 現在できること

### ダッシュボード (`/`)
- データ 0 件時の初回ガイド（回線追加 / 確認用サンプルデータ投入 / 履歴確認 / バックアップ復元）
- 危険案件サマリー（期限超過件数）
- 契約終了が近い回線アラート（30日以内、最大5件）
- 今後のアクション予定（予定日が60日以内または超過の利用中 / 解約予定回線）
- 番号・無料オプション期限アラート（MNP予約番号期限 / 無料オプション期限が3日以内または超過、最大5件）
- 特典期限アラート（未受取かつ受取期限日が30日以内または超過、最大5件）
- 長期未活動の回線カード（90日基準、最大5件）＋「活動を記録」クイックリンク
- 近日期限集計（1〜3日 / 4〜7日 排他表示）
- 状態別件数（利用中 / 解約予定 / 登録総数）
- 月額費用サマリー（設定済み回線の合計）
- `BenefitRecord` ベースの概算 `収支サマリー`（累計支払コスト / 受取済み特典 / 実質収支）
- 光回線向けの `残債解消予定日 / あとN日 / 概算残債` ダッシュボード補助カード
- `利用中` / `解約予定` 回線の `通 / 話 / S` 利用実績不足を巡回できるダッシュボード補助カード
- 左側サイドバーは、項目が増えてもヘッダーを残したまま nav だけ縦スクロール可能
- 通知方針サマリー・通知対象件数・通知理由別件数
- 通知対象回線一覧 → `/lines` 理由付き遷移

### 主台帳 (`/lines`)
- 回線の追加・編集・削除・Undo（Ctrl+Z / ⌘Z）
- データ 0 件時の空状態ガイド（回線フォーム / 確認用サンプルデータ投入 / バックアップ復元 / 履歴ページ）
- 電話番号全文保存・原則マスク表示
- 回線種別 / 月額費用 / 次回確認日 / 契約開始日 / 契約終了日
- 今後のアクション予定日 / 今後のアクション種別 / 次に移るキャリア
- MNP予約番号 / MNP予約番号の有効期限 / 無料オプション期限
- `LineDraft.benefits: BenefitRecord[]` による特典 / キャッシュバック管理
- 特典の追加・編集・削除（種別 / 金額 / 受取期限日 / 受取条件 / 受取済み / 受取日 / メモ）
- 契約者 / 使用者 / 支払方法 / プラン / 端末 / 契約名義メモ
- `lineType === 光回線` のときだけ、移行種別 / ISP 名 / 工事費関連を構造化して保存・表示
- 名義が複数あるときだけ、ダッシュボードに `名義別サマリー` を表示
- キーワード・契約状態・回線種別による絞り込み
- 並び替え（次回確認日 / 月額費用 / 作成日時 / 最終活動日）
- URLパラメータで並び順を指定可（`?sort=latestActivityAsc` など）
- 複数選択・一括ステータス変更・一括削除
- 行ごとの詳細表示の開閉
- 契約開始日から 181 日後の `解約可能推奨日` を自動表示
- 詳細で `今後のアクション予定日` / `今後のアクション種別` / `次に移るキャリア` を表示
- 詳細で `MNP予約番号` / `MNP予約番号の有効期限` / `無料オプション期限` を表示
- 詳細で `特典 / キャッシュバック` の件数・受取状況・受取条件・メモを表示
- `利用中` / `解約予定` の回線で、180日以内の `通 / 話 / S` 利用実績バッジを表示
- 行ボタンから「活動を記録」クイックアクション

### 契約履歴・活動ログ (`/lines/history`)
- 履歴エントリの登録・編集・削除（電話番号単位）
- データ 0 件時の空状態ガイド（履歴フォーム / 確認用サンプルデータ投入 / 回線一覧 / バックアップ復元）
- 1件の履歴に複数の活動ログ
- 活動種別のクイック選択ボタン（よく使う種別 / 定義済み種別）
- 活動種別に応じた活動メモ候補（種別別の頻出文言 + fallback 候補）
- 活動メモのクイック候補（固定候補 / 定型候補 / 最近使った文言 / 非表示候補）
- 現在の活動メモ文言をその場で custom 候補として追加・削除・更新
- custom 候補を `上へ` / `下へ` で並び替え可能
- 各候補セクションを折りたたみ可能で、表示状態を再読み込み後も保持
- 候補は重複表示を避けて整理され、現在値と完全一致する候補は選択中表示
- `固定候補` / `非表示候補` / 候補管理全体を 1 操作で初期化可能
- 電話番号に一致する主台帳候補 / 直近履歴候補のワンタップ反映
- 活動日のクイック入力（今日 / 契約開始日 / 前回活動日）
- 活動記録保存後に「次回確認日を更新しますか？」サジェスト（活動日+設定日数）
- 電話番号単位のタイムライン（期間切替・表示対象切替）
- 履歴 JSON エクスポート / インポート
- 主台帳＋契約履歴の統合バックアップエクスポート / インポート
- 履歴入力フォームの下書きを localStorage に自動保存し、再読み込み後に復元
- 復元した下書きを `破棄して新規入力` でその場で捨てられる

### 設定 (`/settings`)
- `/settings/storage` / `/settings/backup` / `/settings/notifications` / `/settings/activity-types` のサブルート導線
- 通知方針設定（対象期限・再通知ポリシー）
- 永続ストレージ状態・schemaVersion・保存件数・最終更新時刻
- 統合バックアップのエクスポート / インポート

## セットアップ

```bash
npm install
npm run dev
```

## 開発用バッジ

作業ブランチを視認するため `.env.local`（gitignore済み）に設定する：

```
VITE_DEV_LABEL="DEV / feat/xxx / PR #NNN"
```

未設定時はバッジ非表示。ファイルはコミット不要・ブランチ変更のたびに手動更新。

## GitHub Actions

- `Repo sanity`: 最低限のファイル存在確認
- `CI`: `npm ci` → `npm run check` → `npm run build`
- `main` 保護: PR 必須 / approval 任意 / required check `check-and-build`

## 確認手順

1. `npm run dev` でサーバー起動
2. `/` でダッシュボードの各カードが表示されることを確認
3. localStorage 空の状態で、`/` に初回ガイド、`/lines` と `/lines/history` に空状態導線が出ることを確認
4. 初回ガイドの `確認用サンプルデータを読み込む` で、主台帳と履歴が即時反映されることを確認
5. `/lines` で回線の追加・編集・検索が動作することを確認
6. `/lines/history` で履歴記録・タイムライン・契約候補反映・活動種別 / 活動メモ / 活動日のクイック入力が動作することを確認
7. `活動メモ` 候補の重複が整理され、現在値と完全一致する候補が選択中表示になることを確認
8. `活動メモ` 候補の `固定` / `固定解除` / `非表示` / `戻す` が再読み込み後も保持されることを確認
9. `この文言を候補に追加` で custom 候補が増え、`現在の文言で更新` と `削除` が動くことを確認
10. custom 候補の `上へ` / `下へ` で表示順が変わり、再読み込み後も維持されることを確認
11. 候補セクションの `折りたたむ` / `展開` が動き、再読み込み後も状態が維持されることを確認
12. 履歴フォームを途中まで入力して再読み込みすると、入力内容が復元されることを確認
13. 復元通知の `破棄して新規入力` を押すとフォームが空に戻り、再読み込みしても下書きが戻らないことを確認
14. `入力をリセット` または `履歴を保存する` 後に再読み込みして、下書きが消えていることを確認
15. `固定候補をクリア` / `非表示候補をクリア` / `候補管理を初期化` が動き、再読み込み後も維持されることを確認
16. `/settings/storage`、`/settings/backup`、`/settings/notifications`、`/settings/activity-types` が直接開けることを確認
17. `/lines` で `今後のアクション予定日` / `今後のアクション種別` / `次に移るキャリア` を入力して保存し、詳細表示に反映されることを確認
18. `/lines` で `MNP予約番号` / `MNP予約番号の有効期限` / `無料オプション期限` を入力して保存し、詳細表示に反映されることを確認
19. `/lines` で特典を追加・編集・削除し、詳細表示に `特典 / キャッシュバック` が反映されることを確認
20. 旧 `schemaVersion: 7` の主台帳データを読み込んでも、光回線固有フィールドがデフォルト値で補完されて表示が壊れないことを確認
21. `/lines` で `回線種別 = 光回線` を選ぶと、移行種別 / ISP 名 / 工事費関連のフィールドが表示されることを確認
22. 光回線の詳細表示に `残債解消予定日` と `概算残債` が表示されることを確認
23. `/` の `特典期限アラート` に、未受取かつ受取期限日が30日以内または超過の特典が表示されることを確認
24. `/` の `今後のアクション予定` に、予定日が60日以内または超過の利用中 / 解約予定回線が表示されることを確認
25. `/` の `番号・無料オプション期限` に、MNP予約番号期限と無料オプション期限が3日以内または超過の回線が表示されることを確認
26. `利用中` または `解約予定` の回線詳細に `解約可能推奨日` が表示され、`解約済み` / `MNP転出済み` では表示されないことを確認
27. `利用中` または `解約予定` の回線に `通 / 話 / S` の利用実績バッジが表示され、実績ゼロの回線では `利用実績なし` が表示されることを確認
28. 契約者名義が複数あるデータでは `/` に `名義別サマリー` が表示され、名義が1種類だけのデータでは表示されないことを確認
29. `/` に `収支サマリー` が表示され、`累計支払コスト / 受取済み特典 / 実質収支` が概算として確認できることを確認
30. sample data を読み込んだ `/` に光回線向けカードが表示され、`残債解消予定日 / あとN日 / 概算残債` が確認できることを確認
31. `/` に `利用実績種別の巡回` が表示され、`通 / 話 / S` のどれが不足しているかが確認できることを確認
32. 高さの低い viewport でも、左側サイドバーのタブ群が縦スクロールできることを確認
33. `npx tsc --noEmit` でビルドエラーがないことを確認
34. PR では GitHub Actions の `CI` workflow が成功していることを確認

## 重要方針

- 秘密情報は repo に保存しない
- 暗号化本体・通知本実装・IndexedDB 本体は段階的に進める
- AI が実装ドライバー、人間がプロダクトオーナー（Issue/PR で自然言語指示）
