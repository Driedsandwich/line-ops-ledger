# 回線運用台帳

ローカル優先・暗号化前提の PWA として、回線・期限・証跡・特典を安全に管理するためのリポジトリです。

## 現在できること
- `/` で保存済み回線の実データから危険案件と近日期限を集計表示できる
- 危険案件が 0 件でも `問題なし` を表示できる
- `/lines` で回線ドラフトを追加・編集・削除できる
- `/lines` で次回確認日を保持できる
- 直前の追加・更新・削除を `操作を戻す` または `Ctrl+Z` / `⌘Z` で戻せる
- `/settings` で永続ストレージ状態、schema version、保存件数、最終更新時刻、保存形式を確認できる
- `/settings` で JSON バックアップのエクスポート / インポートができる
- manifest / service worker / offline shell を確認できる

## セットアップ
```bash
npm install
npm run dev
```

## 確認手順
1. ブラウザで `/settings` を開く
2. schema version、保存件数、最終更新時刻、保存形式が表示されることを確認する
3. `JSON をエクスポート` を押し、JSON ファイルが出力されることを確認する
4. `JSON をインポート` から同じファイルを読み込み、件数が維持されることを確認する
5. `/lines` と `/` の既存機能が壊れていないことを確認する

## 現在の優先事項
- 次の PR: 保存層の差し替えを見据えて、正式スキーマと永続化単位を整理する
- 直近の Issue: JSON バックアップのエクスポート / インポート

## 重要方針
- 秘密情報は repo に保存しない
- docs-only では終えず、毎回確認可能な挙動差分を出す
- 暗号化本体・通知本実装・IndexedDB 本体は段階的に進める
